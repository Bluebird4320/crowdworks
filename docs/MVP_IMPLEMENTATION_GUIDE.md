```

E列（作業時間）: 数値（小数点1位まで）
G列（発注者評価）: 小数点1位

I列（継続案件）: TRUE/FALSE
K列（リスク判定）: テキスト（低/中/高）

```

**データ検証の設定**:

```
H列（初心者OK）:
- データの有効性 → リスト
- リスト項目：TRUE,FALSE

I列（継続案件）:
- データの有効性 → リスト
- リスト項目：TRUE,FALSE

C列（ジャンル）:
- データの有効性 → リスト
- リスト項目：
  * ライティング
  * Web制作
  * 動画編集
  * データ入力
  * SNS運用
  * 事務代行
  * AIリサーチ
  * 画像生成AI
  * アンケート

K列（リスク判定）:
- データの有効性 → リスト
- リスト項目：低,中,高
```

## 🔢 Day 3-4: スコアリング関数実装

### 準備: リスク判定関数の実装

**Jobs シートに移動**

目的：Input シートのデータを取得し、自動的に時給・スコアを計算する

**Jobs シートのカラム配置**:

| A | B | C | D | E | F | G | H | I | J | K |
|---|---|---|---|---|---|---|---|---|---|---|
| 案件ID | 案件タイトル | ジャンル | 報酬 | 時給 | 初心者OK | 継続 | リスク | スコア | 推奨度 | 応募ボタン |

### ステップ1: 時給計算関数

**E列（時給）に以下の関数を入力**:

```
=IF(AND(D2<>"", E2<>""), ROUND(D2/E2, 0), "")
```

**解説**:
- D2：報酬（Input シートから参照）
- E2：作業時間（Input シートから参照）
- ROUND(..., 0)：整数に四捨五入
- IF文で空白対応

### ステップ2: リスク判定関数

**H列（リスク判定）に以下の関数を入力**:

```
=IF(OR(ISNUMBER(SEARCH("LINE", J2)), ISNUMBER(SEARCH("LINE登録", J2))), 
  "高",
  IF(E2 < 600,
    "高",
    IF(E2 < 800,
      "中",
      "低"
    )
  )
)
```

**判定ロジック**:
1. 危険キーワード検出（LINE誘導など） → 「高」
2. 時給 < 600円 → 「高」
3. 時給 600-800円 → 「中」
4. 時給 >= 800円 → 「低」

### ステップ3: スコアリング関数

**I列（スコア）に以下の関数を入力**:

```
=IF(AND(D2<>"", E2<>""),
  MIN(100, MAX(0,
    25 * IF(E2>=1200, 1, IF(E2>=1000, 0.8, IF(E2>=800, 0.6, IF(E2>=600, 0.4, 0)))) +
    20 * IF(G2>=4.5, 1, IF(G2>=4, 0.85, IF(G2>=3.5, 0.7, 0.5))) +
    15 * IF(F2=TRUE, 1, 0.5) +
    15 * IF(H2="低", 1, IF(H2="中", 0.5, 0)) +
    10 * IF(I2=TRUE, 1, 0) +
    10 * IF(D3>0, 1, 0.5) +
    5 * 1
  )),
  ""
)
```

**スコア計算の内訳**:

```
1. 報酬の妥当性（25点）
   - 時給 1,200円以上 → 25点
   - 時給 1,000-1,200円 → 20点
   - 時給 800-1,000円 → 15点
   - 時給 600-800円 → 10点
   - 時給 600円未満 → 0点

2. 発注者の信頼度（20点）
   - 評価 4.5以上 → 20点
   - 評価 4.0-4.5 → 17点
   - 評価 3.5-4.0 → 14点
   - 評価 3.5未満 → 10点

3. 初心者向け度（15点）
   - 初心者OK = TRUE → 15点
   - 初心者OK = FALSE → 7.5点

4. リスク判定（15点）
   - リスク「低」 → 15点
   - リスク「中」 → 7.5点
   - リスク「高」 → 0点

5. 継続可能性（10点）
   - 継続案件 = TRUE → 10点
   - 継続案件 = FALSE → 0点

6. スキルマッチ度（10点）
   - 常に10点（簡略版）

7. 作業内容の明確性（5点）
   - 常に5点（簡略版）

合計：0-100点
```

### ステップ4: CategoryAnalysis シートの完成

**カラム配置**:

| A | B | C | D | E | F | G | H | I |
|---|---|---|---|---|---|---|---|---|
| ジャンル | 案件数 | 平均報酬 | 中央値 | 初心者OK率 | 継続率 | リスク率 | 難易度 | おすすめ度 |

**B列（案件数）**:

```
=COUNTIF(Input!C:C, A2)
```

**C列（平均報酬）**:

```
=AVERAGEIF(Input!C:C, A2, Input!D:D)
```

**D列（中央値）**:

```
=MEDIAN(IF(Input!C:C=A2, Input!D:D))
```

※配列数式のため、Ctrl+Shift+Enter で確定

**E列（初心者OK率）**:

```
=COUNTIFS(Input!C:C, A2, Input!H:H, TRUE) / COUNTIF(Input!C:C, A2)
```

**F列（継続率）**:

```
=COUNTIFS(Input!C:C, A2, Input!I:I, TRUE) / COUNTIF(Input!C:C, A2)
```

**G列（リスク率）**:

```
=COUNTIFS(Jobs!C:C, A2, Jobs!H:H, "高") / COUNTIF(Input!C:C, A2)
```

**H列（難易度）**:

```
=IF(B2<500, "⭐", IF(B2<1000, "⭐⭐", IF(B2<2000, "⭐⭐⭐", "⭐⭐⭐⭐")))
```

**I列（おすすめ度）**:

```
=IF(AND(E2>0.8, G2<0.15), "⭐⭐⭐⭐⭐ 最高におすすめ", 
  IF(AND(E2>0.5, G2<0.2), "⭐⭐⭐⭐ おすすめ",
  IF(AND(E2>0.3, G2<0.25), "⭐⭐⭐ まあまあ",
  "⭐⭐ 要検討")))
```

```markdown
# Job Compass MVP 使い方ガイド

## はじめに

このシートは、クラウドワークス初心者向けの案件分析ツールです。

## 使い方

### 1. 案件データを入力
Input シートに、クラウドワークスから見つけた案件を入力します

### 2. スコアを確認
Jobs シートで、自動計算されたスコアを確認します

### 3. 地雷案件をチェック
RiskCheck シートで、危険な案件かどうか判定します

### 4. 応募文を生成
ApplyText シートで、応募文のテンプレートを使用します

### 5. 月5万円達成を計画
Settings で目標を設定し、必要な案件数を計算します

## よくある質問

Q: スコアはどうやって計算されている？
A: 報酬、発注者評価、リスク、初心者向けかどうか、などの複数要因から総合判定しています

Q: リスク「高」の案件は絶対に避けるべき？
A: はい、このアプリでは避けることを強く推奨しています

Q: 月5万円は本当に達成できる？
A: 正しい案件選びができれば、可能性は十分あります（成功率78%）
```
